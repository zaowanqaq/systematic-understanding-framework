# 系统性理解框架

> **22 个思维工具。4 个阶段。1 套框架。** 对任何复杂事物建立深度、结构化理解 — 从 Git 原理到量子物理到商业策略。

---

## 🤔 你的痛点

面对一个复杂话题，你读文章、看视频、问 AI——但理解始终浮于表面。你会**用**这些概念，但**讲不清楚**；你能跟教程走，但无法从第一性原理推导。

大多数人停在这里。这个框架帮你跨过去。

## 🧠 解决方案

基于 Mauricio Heck（Polymath Investor）的 **「How to Understand Anything」** 方法论，将 **22 个经过验证的思维工具** 组织为四个递进阶段：

| 阶段 | 你要做什么 | 核心工具 |
|------|-----------|----------|
| **1. 拆解与锚定** | 把复杂拆到不可否认的基本单元 | 第一性原理、分解、帕累托锁定、结构化映射、抽象层次切换 |
| **2. 连接与模式化** | 把新知挂到你已有的知识结构上 | 类比映射、模式组块、系统原型、参照类推理、区分信号与噪音 |
| **3. 提问与重构** | 挑战假设，找到更好的问题 | 逆向思维、五个为什么、苏格拉底式提问、重新框定、挑战隐藏假设 |
| **4. 整合与验证** | 通过输出检验你的理解 | 心智模型格栅、抽象上升、整合思维、假设检验、证伪、费曼输出 |

---

## 🔥 真的有效吗？

有。同一人学 **Git**——先不用框架，后用框架：

| | 没用框架 | 用了框架 |
|---|---|---|
| **学习者说** | *"你说的我没懂。"* *"还是太抽象了。"* | *"不错不错！"* |
| **为什么** | 在心智模型之前就先教了命令。没有统一结构。 | 第一性原理先行。四区建模。逆向思维。费曼收尾。 |

> **同一人。同一话题。同一个 AI。唯一区别：一套四阶段管道，把信息倾倒变成了理解构建。**

## 🔬 使用场景

- **学一门新技术**（Git、Kubernetes、Rust） — 先建心智模型，再把命令映射到模型上
- **分析一个商业问题** — 分解 → 帕累托锁定 → 逆向思维找失败模式
- **评估一个有争议的话题** — 苏格拉底式提问 + 证伪思维 + 参照类推理
- **教别人或写文章** — 费曼输出暴露你自己的理解缺口
- **做高风险决策** — 心智模型格栅 + 挑战假设 + 逆向思维

## 🤖 各 Agent 接入方法

让任何 AI 编程助手变成系统性思考者。选择你的平台：

### Trae / Codex

把 skill 文件放入技能目录：

```bash
cp systematic-understanding.skill .trae/skills/
```

重启或重载窗口即可。当你让 Agent 解释、分析或帮你理解某个东西时，它会自动调用这个框架。

### Claude Code

添加到 CLAUDE.md 或 skills 目录：

```bash
cp -r systematic-understanding/ .claude/skills/
```

或者直接粘贴提示词：

```
当帮我理解任何事物时，使用系统性理解框架：
1. 拆解与锚定（第一性原理、分解）
2. 连接与模式化（类比、系统原型）
3. 提问与重构（逆向思维、五个为什么、苏格拉底式提问）
4. 整合与验证（费曼输出、跨领域连接）
```

### Cursor

添加到 `.cursor/rules/`：

```bash
cp SKILL.md .cursor/rules/systematic-understanding.mdc
```

### GitHub Copilot

添加到 `.github/copilot-instructions.md`：

```markdown
在帮我学习时使用系统性理解框架：
- 先讲第一性原理，再给解决方案
- 建立心智模型，不只是列命令
- 多用类比、逆向思维和费曼技巧
```

### Windsurf

添加到 `.windsurfrules`：

```
解释概念时使用四阶段系统性理解框架：
拆解 → 连接 → 重构 → 验证。心智模型优先于步骤教学。
```

### 任意 Agent（手动）

直接告诉你的 Agent：

> *"帮我理解东西的时候，用 github.com/zaowanqaq/systematic-understanding-framework 里的系统性理解框架。"*
